運彩：後台球版功能：注單計算

運彩：後台球版功能

參考連結

  1. 在投注單頁面，點擊「下注」
    • 下注失敗
      • 帳戶餘額不足
      • 帳號啟用狀態「停用」
    • 下注成功時，跳「下注處理」
  2. 進行下注處理
    • 檢查賽事狀態，若為以下狀態時，為投注失敗
      • 取消 / 暫停投注 / 結算
    • 檢查返獎率，若為以下狀態時，為投注失敗
      • 賽前 變為 走地
    • 檢查賠率變動，若為以下狀態時，為投注失敗
      • 任一盤口賠率，下注時盤口賠率 - 當前盤扣賠率，並取絕對值後大於 20 的話
    • 預扣成功投注的注單金額
  3. 下注處理結果頁面
    • 成功筆數（注單成立，注單狀態「待處理」）
    • 失敗筆數（賽事兩隊伍，賽事開始時間、投注對象、失敗原因）
      • 賽事狀態為以下時，顯示對應的文字內容
        • 賽事狀態「取消」，文字內容為：「賽事已被取消」
        • 賽事狀態「暫停投注」，文字內容為：「賽事已暫停投注」
        • 賽事狀態「結算」，文字內容為：文字內容為：「賽事已結算」
      • 返獎率 或 盤扣賠率 變更過大時，顯示對應的文字內容
        • 文字內容為：「賽事賠率變化幅度過大」
    • 按鈕「回到比賽列表」
  4. 等待「取消注單」時限
    • 取消下注
      • 點擊「取消下注」時，跳出提示訊息「是否確定取消注單」
      • 點擊「確認」時，跳出提示訊息「已取消注單」（變更注單狀態為「取消」）
    • 「取消注單」時限已到（確認注單下注，變更注單狀態為「待結算」）
  5. 「待結算」注單
    • 若賽事取消
      • 取消注單
        • 注單欄位，為以下狀態
          • 有效下注 = 0
          • 已派彩金額 = 下注金額
          • 輸贏金額 = 0
          • 注單狀態改為「取消」
        • 發送訊息文字為以下內容
          • 推送通知（顯示於手機通知欄/鎖屏的標題）
            • 【投注通知】您有一筆注單取消！
          • 通知詳情標題（顯示於 App 內通知詳情的標題）
            • 【投注通知】
          • 通知詳情內容（顯示於 App 內通知詳情的內容）
            • 您投注的賽事「主隊 vs 客隊」已取消，注單 xxx 會自動退款，請留意帳戶餘額更新。
    • 若賽事結算，跳「注單結算處理」
  6. 注單結算處理，以下處理順序
    1. 判定注單輸贏局
      • 賽事類型＝讓球
        • 比較主隊、客隊最終進球數
        • 當兩隊進球數相同時，視為輸局
      • 賽事類型＝大小球
        • 比較 大小球球數 與 最終總進球數
    2. 判定是否符合盤口條件
      • 賽事類型＝讓球
        • 兩隊最終進球數，是否符合 讓球數，以下為例
          • A隊 讓 B隊 1分，比分結果為 0 : 1、1 : 2、2 : 3 ...等依此類推
      • 賽事類型＝大小球
        • 盤口大小球球數 = 最終總進球數
      • 若符合盤口條件，使用「盤口賠率」計算金額
    3. 玩家贏局時，通知中心發送以下內容（輸局不發通知）
      • 推送通知（顯示於手機通知欄/鎖屏的標題）
        • 【投注通知】您有一筆注單中獎！
      • 通知詳情標題（顯示於 App 內通知詳情的標題）
        • 【投注通知】
      • 通知詳情內容（顯示於 App 內通知詳情的內容）
        • 恭喜您！🎉
          您投注的賽事「{{主隊名稱}} vs {{客隊名稱}}」已結算，注單編號 {{注單ID}} 已中獎，中獎金額 1,200。
          派彩金額將於稍後派發，請留意系統通知。
    4. 計算返獎金額（玩家「輸局」、「贏局」時）
      • 玩家贏局且不符合盤口條件：有效下注金額 X (1-對應返獎率)
      • 玩家輸局且不符合盤口條件：有效下注金額
      • 玩家贏局且符合盤口條件：有效下注金額 X (1-對應返獎率) X (盤口賠率/100)
      • 玩家輸局且符合盤口條件：有效下注金額 X (盤口賠率/100)
      • 註：
        • 對應返獎率：根據「賽事時間」判斷是使用「賽前」or「賽中」返獎率
    5. 計算佔成支出（玩家「贏局」時）
      • 玩家贏局且不符合盤口條件：有效下注金額 X 對應返獎率
      • 玩家贏局且符合盤口條件：有效下注金額 X 對應返獎率 X (盤口賠率/100)
    6. 計算返佣金額
      • 有效下注金額 x 下層對應返佣率＝返佣金額
        • 下層對應返佣率：依據 SH階層 設定給 SSMA 的返佣率
    7. 將「返佣金額」從「返獎金額」中扣除（返獎金額 - 返佣金額 ＝ 返佣後金額）
      • 若「返獎金額」<「返佣金額」，需要由從 SH 帳戶餘額中扣除
    8. 將「返佣金額」分配給各階層順序為 SSMA > SMA > MA > Agent
    9. 將「返佣後金額」或「佔成支出」分配給各階層順序為 SH > SSMA > SMA > MA > Agent
    10. 計算派彩金額
      • 玩家贏局且不符合盤口條件：有效下注金額 X 對應返獎率
      • 玩家輸局且不符合盤口條件：有效下注金額
      • 玩家贏局且符合盤口條件：有效下注金額 X (盤口賠率/100) X 對應返獎率
      • 玩家輸局且符合盤口條件：有效下注金額 X (盤口賠率/100)
      • 註：
        • 派彩金額必為大於等於0
        • 注單狀態為「取消」時，派彩金額 = 下注金額
        • 對應返獎率：根據「賽事時間」判斷是使用「賽前」or「賽中」返獎率
    11. 計算「輸贏金額」
      • 派彩金額 - 有效下注金額
      • 註：
        • 注單狀態為「取消」時，輸贏金額 = 0
    12. 系統進行派彩
      • 發送派彩金額至會員帳戶餘額，跳「注單狀態「已結算」」
      • 發送信息至「通知中心」，以下為發送訊息之內容
        • 玩家贏局時
          • 推送通知（顯示於手機通知欄/鎖屏的標題）
            • 【投注通知】您有一筆注單已派彩！
          • 通知詳情標題（顯示於 App 內通知詳情的標題）
            • 【投注通知】
          • 通知詳情內容（顯示於 App 內通知詳情的內容）
            • 注單 xxx 已完成派彩 ，派彩金額 1,200
        • 因賽事取消：
          • 推送通知（顯示於手機通知欄/鎖屏的標題）
            • 【投注通知】您有一筆注單已派彩！
          • 通知詳情標題（顯示於 App 內通知詳情的標題）
            • 【投注通知】
          • 通知詳情內容（顯示於 App 內通知詳情的內容）
            注單 xxx 投注金額 500 已退回至您的帳戶。
  7. 注單狀態「已結算」
