我正在開發一個基於 Golang（後端 API）與 Laravue（前端）的後台網站，目前已有設計完成的會員列表畫面。

目前已知條件如下：

  • 後端機器 IP 為：106.104.110.60
  • 後端使用 Golang 撰寫 HTTP Server，採用 GET / POST 的 RESTful API，資料皆為 JSON 格式
  • 與後端串接時，預設使用 HTTP/2 技術（至少 HTTP/1.1 以上）
  • MySQL port 為 3306，但目前不需要使用資料庫
  • 後端 API 假設已實作，不需重複建立，只需在前端程式碼中保留串接點與 axios 實作註解

🟡 預期功能：
我想要建立一個「會員列表」管理頁面，欄位如下：

  • 帳號（username）
  • 姓名（name）
  • 上層代理（agent_name）
  • 餘額（balance）

🟢 前端技術條件：

  • 前端框架為 Vue，UI 套件為 Laravue（Element UI）
  • 請以 Laravue 組件風格撰寫前端頁面與元件結構
  • 使用 el-table 顯示會員資料
  • 新增與刪除按鈕功能可為空，但需保留事件鉤子與註解
  • 使用 axios 串接後端假設的 API（路徑為：http://106.104.110.60/api/members），並確保支援 HTTP/2 串接（axios 預設支援，請於註解中說明）

🔧 請產出以下內容：

  1. 完整的 Vue 組件檔案格式，含 template、script、style
  2. 資料初始化為本地假資料，但在 mounted 中保留 axios 串接後端的註解範例（不需實際呼叫）
  3. 在按鈕功能中留有註解與可擴充事件處理器（如 handleAdd / handleDelete）
  4. 程式碼需可直接貼入 Laravue 組件並運作（假資料即可看到畫面）

🔎 附加說明：

  • 請特別注意，後端請完全視為「已完成」
  • 你只需專注在前端代碼架構與資料顯示邏輯即可
  • 串接範例中請在註解中說明：axios 預設支援 HTTP/2（無需特殊設定，但可強調兼容性）

運彩後台規格書生成提示詞

請依照以下規範，撰寫一份「運彩後台」的系統規格書：

  • 文件格式使用 .md (Markdown)，所有內容統一寫在一個檔案中，不要分開。

  • 使用標題層級清楚劃分：

    • 功能大項（如：「權限管理」、「會員管理」等）請用 # (h1) 標題。
    • 該功能底下的子功能（如：「修改密碼」、「管理帳號」等）請用 ## (h2) 標題。

  • 每一個子功能 (##) 內，要包含兩個區段：

    1. 欄位名稱列表
      使用無序列表（-）列出資料庫相關的欄位名稱，每個欄位後面可以加上簡單描述（選填）。
    2. 操作說明表格
      使用 Markdown 的 table 格式，包含以下欄位（由左而右）：
      • 功能操作
      • 執行時影響的欄位
    - 表格舉例：
	| 功能操作         | 執行時影響的欄位             |
	|--------------|-------------------------------|
	| 新增會員     | 姓名、電話、Email、帳號狀態   |
	| 變更狀態     | 帳號狀態                       |

  • 其他注意事項：

    • 所有功能描述需明確且簡潔。
    • 不要漏掉任何一個子功能。
    • 如果某些子功能沒有操作，只列欄位即可，不必硬加表格。
    • 僅依照提供的高保真原型圖推測功能，不要自行增加或刪減功能。
    • 撰寫過程中禁止出現「本系統」、「此平台」等模糊稱呼，直接使用具體功能名稱描述。
