運彩：後台球版功能：權限階層規則

運彩：後台球版功能

層級規則：

  • 使用者層級分為：Root、SH、SSMA、SMA、MA、Agent
  • 各層級代理帳號僅可檢視、修改、停權其下層級的帳號，無法查看同層級或上層級帳號
  • 各層級帳號僅可建立下一層級的帳號，不得越級建立帳號
  • 各層級可使用的功能權限可能不同，實際功能權限配置仍在規劃中
  • Root 為系統最高權限帳號，專供開發／管理團隊內部使用，不對外開放
  • SH 層級使用者無法察覺 Root 層級帳號的存在，即無任何途徑查詢或顯示 Root 帳號相關資訊
  • op 為營運帳號，作為各層級代理帳號的「分身」，其建立與權限配置皆由所屬代理層級帳號控制
  • Member 為前台使用者帳號，屬於一般會員類型，僅由 Agent 層級帳號建立與管理

以下舉例，可參考附件：權限階層圖

  • 各層級代理僅能檢視與管理其下層級帳號，無法查看同層級或其他上層帳號
    • 例：
      • SMA 1 檢視和管理 MA 1、MA 2、agent 1、 agent 2、會員1、會員2、Agent_op
      • SMA 2 只看得到 MA3，看不到和無法操作 SMA 1 底下階層的資料
  • 各層級帳號僅可建立下一層級的帳號，不得越級建立帳號
    • 例：
      • Admin_SH 只能建 SSMA 1 、SSMA 2，如果要建立 SMA ，則需要在後台登入 SSAM 的帳號
      • Admin_SH 只能建 SH_op，不能幫 SSMA 1 建立 SSMA_op
