iOS 足球運彩應用程式原型 - 提示詞

目標

開發一款 iOS 足球運彩應用程式 的 高保真原型介面，並在 類似手機螢幕的網頁 上顯示，確保 HTML 程式碼最少，且每個界面可單獨優化。

需求

1. 功能分析

確保原型涵蓋 比賽瀏覽、下注、結果查詢、個人中心 等核心功能。

2. 界面規劃

設計 10 個主要頁面，例如：

  • 首頁（熱門賽事）
  • 比賽列表（足球項目篩選）
  • 下注頁面（投注選項、金額輸入）
  • 投注單（投注詳情確認）
  • 比賽結果（比分與中獎資訊）
  • 個人中心（餘額、交易記錄）
  • 通知與消息中心（賽事提醒）

3. 技術要求

  • HTML 拆分：每個界面獨立成 page1.html ~ page10.html
  • index.html 應平鋪顯示所有界面，而不是提供跳轉連結
  • UI 設計：
  • 類似 iOS 介面（圓角、流暢動畫、原生按鈕）
  • 使用真實圖片與 FontAwesome 圖標，避免佔位符
  • 自適應 iPhone 螢幕尺寸（如 430px 寬的 iPhone 14 Pro）
  • 減少 HTML 行數，透過 CSS（如 Tailwind CSS）與 JavaScript 優化
  • 載入方式：
  • index.html 透過 <iframe> 或 fetch 動態載入 page1.html ~ page10.html

餘額與投注紀錄

  • Bet record  → 進入投注紀錄頁面
  • Balance：顯示顯示用戶餘額

  • 顯示當前比賽或系統維護通知

  • 設定 → 進入設定頁面

  • 左側返回按鈕

  • 右上角導航至「比分數據頁面」

日期分類

  • 顯示 yesterday results 與 today YYYY-MM-DD
  • 點擊可展開/收起日期選單
  • 每個分類下列出賽事清單

賽事分類

  • 顯示 Europa League，Europa Conference League 等分類
  • 每個分類下列出比賽清單

比賽列表 (動態 API 生成)

  • 球隊名稱
  • 比賽時間
  • 投注數據 (紅色/黑色數值)

  • 已選投注顯示區塊
