運彩：後台球版功能：注單管理

緬甸球版/SportsBetting：運彩後台/後台規格書/運彩：後台球版功能

功能說明：

  • 顯示所有玩家注單記錄。
  • 緬甸球版/SportsBetting：運彩後台/後台規格書/運彩：後台球版功能：注單計算
欄位代號 欄位名稱 說明 驗證條件／限制
member_id 會員帳號 會員登入使用的帳號，系統唯一識別標籤 系統自動填入，無法修改
member_name 會員姓名 系統自動填入，無法修改
order_id 注單編號 玩家投注單單號 系統自動填入，無法修改
order_status 注單狀態 待結算、結算中、已結算、取消 系統自動填入，無法修改
待處理：玩家投注後，取消投注的時限還沒到
待結算：賽事還未結束，有效投注 = 0
結算中：賽事狀態「結算」，但還未派彩，有效投注 = 投注金額
已結算：賽事狀態「結算」，且已派彩，有效投注 = 投注金額
取消：有效投注為 0 ，因「被玩家取消注單」、「賠率過大系統自動取消注單」、「賽事被取消」等狀況時，才會出現此狀態
bet 投注金額 玩家下注金額 系統自動填入前台所投注金額，無法修改，且預扣玩家下注金額
payout 派彩金額 贏局且不符合盤口條件時，所獲得的金額
公式：有效投注金額 x ( 1+ (返獎率))

贏局且符合盤口條件時，所獲得的金額
公式：有效投注金額 x ( 1+ (盤口賠率/100) x (返獎率))

有效投注金額為 0 時
公式：投注金額 = 派彩金額		 系統自動計算後填入，且不會出現負數，無法修改
返獎率有「讓球賽前返獎率」、「讓球走地返獎率」、「大小球賽前返獎率」、「大小球賽中返獎率」
win_lose 輸贏金額 派彩金額 - 有效投注金額

有效投注金額為 0 時
輸贏金額 = 0		 系統自動計算後填入，無法修改
bet_time 投注時間 玩家在前台下注的時間 系統自動填入，無法修改
格式：年月日時分秒
payout_time 派彩時間 系統發放贏局金額的時間 系統自動填入，無法修改
格式：年月日時分秒
game_id 賽事編號 賽事使用編號為系統唯一識別標籤 系統自動填入，無法修改
bet_type 投注類型 分為「讓球」、「大小」球兩種
讓球：依主客隊讓球數，來投注哪一隊獲勝
大小球：依比賽結束時的總進球數，來投注是否大於或小於總進球數		 系統自動填入，無法修改
bet_object 投注對象 賽事類型＝讓球，顯示「主隊」、「客隊」其中之一隊伍
賽事類型＝大小球，顯示「大球」、「小球」其中之一		 系統自動填入，無法修改
game_begin_time 賽事開始時間 比賽開始時間 系統自動填入，無法修改
game_result 投注結果 分為 HL、L、HW、W 等結果

HL = 輸局，且使用盤口賠率
L = 輸局，且未使用盤口賠率
HW = 輸局，且使用盤口賠率
W = 輸局，且未使用盤口賠率，包含賽事為平手時		 填入後台賽事結果後，系統根據注單內容判斷後自動填入，無法修改
valid_bet 有效投注金額 注單成立時，玩家投注金額 = 有效投注金額 系統自動填入，無法修改

注單成立：賽事狀態「結算」
注單不成立：注單狀態「取消」時，有效投注金額 = 0
odds 返獎率 根據「賽事類型」、「投注對象」、「賽事時間」顯示玩家當下投注的返獎率 系統自動填入，無法修改
handicap_odds 盤口賠率 根據「賽事類型」顯示玩家當下投注的盤口賠率 系統自動填入，無法修改
handicap 讓球隊伍 讓球隊伍名稱 必填；需要為主隊名稱或客隊名稱

功能操作說明

功能操作 權限階層限制 操作方式 系統行為說明
查詢管理員資料 所有代理層級 1.根據「條件」輸入關鍵字後，點擊「查詢」按鈕
2.直接點擊「篩選條件」
3.點擊特定頁碼
 顯示系統內所有管理員資料，每頁顯示 25 筆資料
根據「條件」的選單內容：管理員帳號，模糊查詢
若有多個「篩選條件」，採用 AND 邏輯來過濾資料
未設定條件時，預設顯示最近創建時間的資料
查詢詳情 所有代理層級 操作欄位點擊「查詢詳情」
